# 통계 데이터 추출
- 유저별 찜, 작성 리뷰 갯수 추출
- 방법 1 : JPQL 생성자 투사(DTO 직접 조회)
- 방법 2 : 엔티티 클래스에 가상의 필드를 만들어 데이터가 따라다니게 만듬

## 방법 1 
- 차용 이유
    1. 엔티티 순수성 유지 : DB에 존재하지 않는 count컬럼을 엔티티에 넣으면 객체의 책임이 무거워지고 오염. 조회(Query)를 위한 데이터는 조회용 객체(DTO)가 처리하도록 하기 위함
    2. 성능 최적화의 유연성(N+1 문제 원천 차단) : 엔티티에 `List<Wishlist>`를 두고 `.size()`를 호출하면, 유저 수만큼 카운트 쿼리가 나가는 N+1 문제가 터짐. JPQL 생성자(Projection)를 쓰면, 단 한 번의 쿼리로 모든 통계를 긁어올 수 있음.
    3. 어드민 기능의 격리 : 어드민에서만 사용하기 때문

### DTO 수정

```java
package com.Connectedm.backend.domain.admin.dto;

import com.Connectedm.backend.domain.user.entity.UserRole;
import com.Connectedm.backend.domain.user.entity.UserStatus;
import lombok.AllArgsConstructor; // 생성자 조회를 위해 추가! 
import lombok.Builder;
import lombok.Getter;

import java.time.LocalDateTime;

@Getter
@Builder
@AllArgsConstructor // 👈 JPQL에서 'new' 키워드로 바로 꽂아주려면 이게 필요합니다!! 
public class AdminUserResponseDto {
    private Long userId;
    private String email;
    private String nickname;
    private String realName;
    private String phoneNumber;
    private UserRole role;
    private UserStatus status;
    private int reportedCount;
    private LocalDateTime createdAt;
    private LocalDateTime lastLoginAt;
    
    // 프론트 요청 추가 데이터 ㅋ
    private Long wishcount;    // 찜한 영화 개수 
    private Long reviewcount;  // 작성한 리뷰 개수 
}
```

### AdminService 수정
- getAllUsers() 수정
```java
// 기존
public List<AdminUserResponseDto> getAllUsers() {
        return userRepository.findAll(Sort.by(Sort.Direction.DESC, "createdAt"))
                .stream()
                .map(user -> AdminUserResponseDto.builder()
                        .userId(user.getId())
                        .email(user.getEmail())
                        .nickname(user.getNickname())
                        .realName(user.getRealName())
                        .phoneNumber(user.getPhoneNumber())
                        .role(user.getRole())
                        .status(user.getStatus())
                        .reportedCount(user.getReportedCount())
                        .createdAt(user.getCreatedAt())
                        .lastLoginAt(user.getLastLoginAt())
                        .build())
                .collect(Collectors.toList());
    }

// 수정 : findAllUserStats로 교체
public Page<AdminUserResponseDto> getAllUsers(Pageable pageable) {
        // 기존: userRepository.findAll() -> stream.map() (N+1 위험 및 수동 매핑)
        // 변경: 리포지토리의 JPQL을 사용하여 서브쿼리로 한 번에 찜/리뷰 개수 긁어옴
        return userRepository.findAllUserStats(pageable);
    }
```
