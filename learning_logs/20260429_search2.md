🚀 [Connected-M] 시맨틱 검색 엔진 데이터 복구 및 동기화 가이드
본 문서는 DB 초기화 등으로 인해 analysis_cache 테이블의 벡터 데이터가 유실되었을 때, AI 모델을 사용하여 데이터를 다시 채우고 검색 엔진을 정상화하는 과정을 다룹니다.

1. 사전 환경 점검 (Prerequisites)
작업 전, 파이썬 사령부의 독립적인 가상환경이 활성화되어 있어야 합니다.

위치: backend/data_crawling 폴더

가상환경 활성화:

Bash
# Windows PowerShell 기준
.\venv\Scripts\Activate.ps1
필수 라이브러리 설치:

Bash
pip install sentence-transformers pymysql python-dotenv
2. 단계별 실행 프로토콜
STEP 1: AI 벡터 연산 서버 가동
자연어를 좌표(Vector)로 변환해주는 중앙 연산 서버를 먼저 실행합니다.

Bash
python vector_server.py
확인: 터미널에 🌐 벡터 서버가 5000번 포트에서 대기 중입니다! 메시지가 뜨면 성공입니다.

STEP 2: 데이터 주입 및 정제 (DB_Fillter)
DB에 비어있는 embedding_vector와 search_keywords를 AI 모델로 가공하여 채워 넣습니다.

Bash
# 파일명 주의: db_filler가 아닌 db_fillter로 실행
python db_fillter.py
작동 원리:

content 테이블의 영화 줄거리(overview)를 읽어옵니다.

AI 모델(ko-sroberta)이 줄거리를 다차원 숫자 좌표로 변환합니다.

analysis_cache 테이블의 해당 컬럼에 좌표값을 업데이트합니다.

3. 기술적 핵심 개념 (팀 공유용)
시맨틱 검색(Semantic Search): 단순 키워드 매칭이 아니라, 문장의 '의미'를 파악하여 유사한 결과물을 찾아내는 기술입니다.

벡터 임베딩(Vector Embedding): 텍스트를 AI가 이해할 수 있는 숫자의 나열(좌표)로 변환한 것입니다.

코사인 유사도(Cosine Similarity): 사용자의 검색어 좌표와 DB 내 영화 좌표 사이의 각도를 계산하여 "얼마나 의미적으로 가까운가"를 측정합니다.
